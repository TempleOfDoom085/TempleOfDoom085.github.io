// Future Roadmap — The Path to Open World
v6.0 → Equippable inventory, player stats (strength/faith/defense), save system, difficulty modes, more rooms and mini-bosses
Phase 2 → 2D isometric prototype with sprite-based characters, tile movement, animated combat, voiced NPCs
Phase 3 → Full 3D open-world engine — photorealistic medieval fortress, real-time combat, procedural dungeons, multiplayer co-op, orchestral soundtrack

Knights Templar: The Siege of the Undead started as a Python assignment for IT 140 at Southern New Hampshire University. The brief was simple: build a text-based adventure with rooms, items, and win/lose conditions.

What it became is something much larger. Over five versions, the game grew from a terminal script into a feature-rich browser experience with hand-coded SVG artwork, procedural Web Audio chant music, a full combat system, NPC dialogue trees, hidden Bible verse scrolls, historical portraits, and an achievement system culminating in a Saint Michael the Archangel cutscene.

The game is built entirely without external libraries — no game engine, no asset files, no audio downloads. Every sound is synthesized live using the Web Audio API. Every piece of artwork is hand-written SVG code following a strict cinematic dark-fantasy art direction inspired by Diablo and Batman Beyond.

The long-term goal is a full open-world 3D game. This browser version is the foundation — the design document, the lore, and the vision are all already built. The graphics are the next frontier.
